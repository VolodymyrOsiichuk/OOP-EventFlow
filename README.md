# EventFlow

EventFlow — консольна система управління подіями, розроблена як підсумковий міні-проєкт з ООП.

Проєкт демонструє використання багатошарової архітектури, принципів SOLID, патернів проєктування, тестування та поступового розвитку функціональності через ітерації.

---

# Основні можливості

## Управління подіями

- створення подій;
- перегляд списку подій;
- категоризація подій;
- контроль максимальної кількості учасників.

## Реєстрація учасників

- реєстрація на подію;
- перевірка доступності місць;
- захист від повторної реєстрації;
- скасування реєстрації.

## Збереження даних

- збереження стану у JSON;
- відновлення стану після запуску програми;
- асинхронний файловий ввід/вивід.

## Аналітика

- пошук активних подій;
- фільтрація за категоріями;
- сортування за популярністю;
- статистика по категоріях.

---

# Архітектура

Проєкт реалізований за принципом багатошарової архітектури.

```text
Console
↓
Application
↓
Domain
↓
Infrastructure
```

## Domain

Містить:

- сутності предметної області;
- бізнес-правила;
- інтерфейси;
- enum-и;
- Result-патерн.

## Application

Містить:

- DTO;
- сервіси застосунку;
- use cases;
- LINQ-запити.

## Infrastructure

Містить:

- репозиторії;
- JSON persistence;
- файлове сховище;
- мапери для серіалізації.

## Console

Містить:

- консольне меню;
- взаємодію користувача із системою.

---

# Бізнес-правила

Система перевіряє:

1. Подія не може бути створена в минулому.
2. Максимальна кількість учасників повинна бути більшою за нуль.
3. Не можна зареєструватися на завершену подію.
4. Не можна перевищити місткість події.
5. Неможливо зареєструватися двічі на одну подію.
6. Неможливо скасувати неіснуючу реєстрацію.
7. Після скасування реєстрації місце звільняється.
