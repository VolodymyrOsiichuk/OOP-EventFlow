# DEMO

## Мета демонстрації

Показати основний функціонал EventFlow, перевірку бізнес-правил, persistence та архітектурні рішення.

Тривалість: 3–5 хвилин.

---

# Крок 1. Запуск застосунку

Запустити:

```bash
dotnet run --project src/EventFlow.Console
```

Показати головне меню.

---

# Крок 2. Створення події

Обрати:

```text
1. Create Event
```

Створити подію:

```text
Назва: Tech Conference
Категорія: Conference
Місткість: 2
```

Показати повідомлення про успішне створення.

---

# Крок 3. Перегляд подій

Обрати:

```text
4. View All Events
```

Показати:

- Id події;
- назву;
- категорію;
- кількість доступних місць.

---

# Крок 4. Реєстрація учасника

Обрати:

```text
2. Register Participant
```

Зареєструвати:

```text
Ім'я: John Smith
Email: john@test.com
```

Показати успішну реєстрацію.

---

# Крок 5. Негативний сценарій

Спробувати зареєструвати того самого учасника повторно.

Очікуваний результат:

```text
Participant already registered
```

Пояснити, що працює бізнес-правило захисту від дублювання.

---

# Крок 6. Скасування реєстрації

Обрати:

```text
3. Cancel Registration
```

Ввести email учасника.

Показати, що реєстрацію успішно скасовано.

---

# Крок 7. Збереження даних

Обрати:

```text
7. Save Data
```

Показати створення JSON-файлу.

---

# Крок 8. Перезапуск програми

Закрити застосунок.

Запустити повторно.

Система автоматично виконує:

```text
LoadAsync()
```

---

# Крок 9. Перевірка відновлення

Обрати:

```text
4. View All Events
```

Показати, що створена подія була відновлена з JSON.

---

# Крок 10. Пояснення архітектури

Коротко показати структуру проєкту:

```text
Console
↓
Application
↓
Domain
↓
Infrastructure
```

Пояснити:

- Domain містить бізнес-логіку;
- Application містить use cases;
- Infrastructure містить persistence;
- Console відповідає за UI.
