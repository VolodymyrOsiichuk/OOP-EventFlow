# Iteration 2 Report

## Overview

У другій ітерації проєкт EventFlow було розширено від базового вертикального зрізу до прикладного застосунку з підтримкою збереження даних, розширеною бізнес-логікою та аналітичними запитами.

Основною метою ітерації було додати persistence, нові сценарії використання, механізми розширення та підготувати проєкт до етапу поглибленого тестування.

---

## Completed Use Cases

### 1. Create Event

Організатор може створити нову подію із зазначенням:

- назви;
- дати;
- категорії;
- місця проведення;
- максимальної кількості учасників.

### 2. Register Participant

Користувач може зареєструватися на подію.

Під час реєстрації перевіряються бізнес-правила:

- подія повинна бути активною;
- не можна перевищити ліміт учасників;
- заборонено дублювати реєстрацію.

### 3. Cancel Registration

Зареєстрований учасник може скасувати свою реєстрацію до початку події.

Після скасування:

- місце стає доступним для інших учасників;
- учасник видаляється зі списку реєстрацій.

---

## Business Rules

У системі реалізовано такі бізнес-правила:

1. Подія не може бути створена з датою в минулому.
2. Місткість події повинна бути більшою за нуль.
3. Не можна зареєструватися на завершену подію.
4. Не можна перевищити максимальну кількість учасників.
5. Один учасник не може бути зареєстрований двічі.
6. Неможливо скасувати неіснуючу реєстрацію.
7. Після скасування реєстрації місце звільняється.

## Pattern Usage

### Factory Pattern

Використовується для створення подій різних типів.

Переваги:

- централізоване створення об'єктів;
- спрощення логіки створення;
- легке розширення новими типами подій.

У майбутніх ітераціях Factory може бути розширено для підтримки нових категорій або шаблонів подій.
