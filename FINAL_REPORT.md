# FINAL REPORT

# EventFlow

## Загальна інформація

### Назва проєкту

EventFlow

### Тип проєкту

Консольна система управління подіями.

### Версія

v1.0.0

### Мета

Розробити застосунок, який демонструє практичне використання принципів ООП, SOLID, патернів проєктування, persistence, тестування та контролю якості в межах курсу об'єктно-орієнтованого програмування.

---

# Опис предметної області

Система призначена для управління подіями та їх учасниками.

Користувач може:

- створювати події;
- реєструвати учасників;
- скасовувати реєстрації;
- переглядати інформацію про події;
- виконувати пошук і аналіз даних;
- зберігати та відновлювати стан системи.

---

# Архітектурне рішення

Проєкт реалізовано за принципом багатошарової архітектури.

```text
Console
↓
Application
↓
Domain
↓
Infrastructure
```

### Domain

Містить бізнес-сутності та інваріанти.

### Application

Містить use cases та сервіси.

### Infrastructure

Містить persistence і репозиторії.

### Console

Забезпечує взаємодію з користувачем.

# Використані принципи SOLID

## SRP

Виділено окремі сервіси:

- EventService;
- EventQueryService;
- EventPersistenceService.

## OCP

Архітектура дозволяє додавати нові реалізації без зміни існуючого коду.

## LSP

Залежності працюють через абстракції.

## ISP

Використовуються вузькі інтерфейси.

## DIP

Сервіси залежать від контрактів, а не від реалізацій.

# Аналіз продуктивності

Для зберігання реєстрацій використовується:

```text
List<Registration>
```

Причини вибору:

- проста структура;
- невеликий обсяг даних;
- швидке послідовне проходження під час LINQ-запитів.

Для статистики використовується:

```text
Dictionary<EventCategory, int>
```

що забезпечує швидкий доступ за ключем.

Додаткової оптимізації для поточного масштабу системи не потрібно.
